# YouTube Intelligence Engine
### CSCI370  Spring 2026

An end-to-end NLP pipeline that extracts insights from YouTube tech review comments using sentiment analysis, BERTopic topic modelling, hybrid retrieval (BM25 + semantic), and a RAG-based Q&A system powered by an LLM agent.

---

## What It Does

You type a question like *"What comfort issues do users mention about Sony XM6 headphones?"* and the system:
1. Routes the query to the best retrieval strategy (semantic, BM25, or hybrid)
2. Pulls the most relevant real comments from a 24,000+ comment dataset
3. Generates a grounded answer using Qwen2.5-1.5B-Instruct constrained to only use the retrieved evidence
4. Shows you the sentiment breakdown, retrieved comments, and MLflow evaluation metrics

## A Note on `chroma_youtube_rag/` and `mlruns/`

These two folders are included in the repo so the file structure is complete, but you do not need to interact with them directly.

- **`chroma_youtube_rag/`** — the persistent ChromaDB vector database generated by the embeddings notebook. If you re-run the RAG notebook in Colab it will regenerate this. The dashboard does not use this folder  it builds its own in-memory index from `comment_embeddings.npy`.
- **`mlruns/`**  the raw MLflow tracking files from experiment runs. The dashboard reads `mlflow_runs_export.csv` instead, which is the clean exported version of these. You do not need to open or run anything inside `mlruns/`.

## How to Run

### The Notebooks (Colab  run in order)

The notebooks are designed to be run sequentially. Each one produces a CSV that the next one depends on.

**1. Preprocessing**
- Open `csci370_dataset_preprocessing.ipynb` in Colab
- Upload your raw scraped CSV or use the included `youtube_comments_preprocessedfinal.csv`
- Run all cells
- Output: `youtube_comments_preprocessedfinal.csv`

**2. Sentiment Analysis**
- Open `370_project_sentiment_analysis-3.ipynb`
- Upload `youtube_comments_preprocessedfinal.csv`
- Run all cells — RoBERTa inference takes a few minutes, GPU recommended
- Output: `youtube_comments_sentiment.csv`

**3. NER**
- Open `370_project_NER-2.ipynb`
- Upload `youtube_comments_sentiment.csv`
- Run all cells
- Output: `named_entities (2).csv`

**4. BERTopic**
- Open `370_BErt_-2.ipynb`
- Upload `youtube_comments_sentiment.csv`
- Run all cells — BERTopic fitting takes a few minutes
- Output: `youtube_comments_BERTtopics.csv`

**5. Embeddings, RAG & Evaluation**
- Open `embeddings_+_rag_final_shared (3).ipynb`
- Upload `youtube_comments_BERTtopics.csv`
- Mount Google Drive if you want embeddings to persist across sessions
- Run all cells — initial embedding takes ~10 minutes, cached after that
- Outputs: `comment_embeddings.npy`, `mlflow_runs_export.csv`

### The Dashboard (Local)

**Requirements**
- Python 3.9+
- A virtual environment (recommended)
- The following files in the same folder as `app.py`:
  - `youtube_comments_BERTtopics.csv`
  - `comment_embeddings.npy`
  - `mlflow_runs_export.csv`

**Install dependencies**
```bash
pip install streamlit sentence-transformers transformers torch rank-bm25 chromadb pandas numpy
```

**Run the dashboard**
```bash
streamlit run app.py
```

Then open `http://localhost:8501` in Chrome.

> **Note:** The first load takes 1–2 minutes while the embedding model loads and the in-memory index is built. This is cached for the rest of the session.

> **Note on response time:** Answer generation using Qwen2.5-1.5B-Instruct runs on CPU locally and takes approximately 5–15 seconds per query. This is a known limitation of running a 1.5B parameter model without a GPU.
